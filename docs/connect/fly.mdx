# Connect Fly.io

Fly.io runs PostgreSQL close to your app. By default a Fly Postgres app is only
reachable on the private `*.internal` / `*.flycast` network, so connecting from
your machine usually means proxying it out first.

## Option A: proxy the database locally

1. Forward the database to your machine with the Fly CLI:

   ```bash
   fly proxy 5432 -a <your-postgres-app>
   ```

2. In Dora connect to the forwarded port:

   ```text
   postgresql://postgres:<password>@localhost:5432/<db>
   ```

   Get the password with `fly secrets list` / your app's `DATABASE_URL`.

## Option B: a public host

If your database has a public address (a `*.fly.dev` endpoint), paste its
connection string directly. Dora enables SSL for public Fly hosts and skips TLS
for the private `*.internal` / `*.flycast` network.

In Dora press `Mod+Shift+N`, toggle **Use connection string**, paste the URL,
**Test connection**, and **Save**.

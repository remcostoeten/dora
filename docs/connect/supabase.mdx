# Connect Supabase

Supabase is hosted PostgreSQL. Dora connects with a standard connection string
and enables SSL automatically for `*.supabase.co` / `*.supabase.com` hosts.

## With a connection string

1. In the Supabase dashboard, open **Project Settings → Database**.
2. Under **Connection string**, choose the **URI** tab and copy the value. It
   looks like:

   ```text
   postgresql://postgres:[YOUR-PASSWORD]@db.<ref>.supabase.co:5432/postgres
   ```

3. Replace `[YOUR-PASSWORD]` with your database password.
4. In Dora press `Mod+Shift+N`, toggle **Use connection string**, paste the URL,
   **Test connection**, and **Save**.

<Callout type='info'>**Direct vs pooled.** Port `5432` is the direct connection. Supabase also offers a Supavisor pooler (port `6543`, a `pooler.supabase.com` host) for serverless workloads. Either works in Dora; use the direct connection unless you specifically need the pooler.</Callout>

## With your Supabase account

In the connection dialog choose **Connect with Supabase**, sign in, and Dora
lists your projects and builds the connection for you, no string to copy.

SSL is required and is set automatically. See
[connection basics](/docs/connect) for how detection works.

# Task: Regenerate TypeScript Bindings

**Complexity:** Low  
**Recommended agent:** Codex (1 agent) or any small model  
**Branch:** `chore/regen-ts-bindings`  
**Estimated time:** 15 min  

---

## Context

The backend went through 8 refactor phases (commands split, error enum changes, new adapter traits, new commands). The TypeScript bindings file at `apps/desktop/src/lib/bindings.ts` is auto-generated by `tauri-specta` when the app starts in dev mode. It may be out of sync.

The one unchecked item in `docs/backend-refactor-checklist.md` (Phase 8) is:
> Regenerate TypeScript bindings after all backend changes

---

## What to do

### Step 1 ΓÇö Verify current bindings are stale

Check the `BackendErrorShape` / `Error` type. After Phase 2-3, the backend serialises errors as:
```json
{ "kind": "ConnectionNotFound", "detail": "connection abc-123 not found" }
```

The frontend utility at `apps/desktop/src/shared/utils/backend-error.ts` defines:
```ts
export type BackendErrorShape = { kind: string; detail: string }
```

Check `apps/desktop/src/lib/bindings.ts` for how `Error` variants appear. If you see `{ name: string; message: string }` or raw Rust enum variants instead of `{ kind, detail }`, the bindings are stale.

### Step 2 ΓÇö Trigger regen

**Option A (preferred):** Run the Tauri dev build. `tauri-specta` rewrites `bindings.ts` on startup.

```bash
cd apps/desktop
bun tauri:dev
# Wait for "Finished" in the Rust compile output, then Ctrl+C immediately after
# bindings.ts is written at startup before the window opens
```

**Option B (if dev mode unavailable):** Run cargo build and check if specta output can be generated standalone:
```bash
cd apps/desktop/src-tauri
cargo build 2>&1 | tail -20
```

### Step 3 ΓÇö Verify the output

After regen, confirm in `apps/desktop/src/lib/bindings.ts`:

1. `Error` type matches `{ kind: string; detail: string }` (not old shape)
2. All commands from `database/commands/` are present:
   - `get_database_schema` ΓåÆ `getDatabaseSchema`
   - `get_database_metadata` ΓåÆ `getDatabaseMetadata`
   - `execute_mutation` family
   - `start_live_monitor` / `stop_live_monitor`
3. Types present: `DatabaseSchema`, `TableInfo`, `ColumnInfo`, `ForeignKeyInfo`, `IndexInfo`

Run TypeScript check:
```bash
cd apps/desktop
~/.bun/bin/bun x tsc --noEmit -p tsconfig.app.json
```
Must exit 0.

### Step 4 ΓÇö Commit

```bash
git checkout -b chore/regen-ts-bindings
git add apps/desktop/src/lib/bindings.ts
git -c commit.gpgsign=false commit -m "chore: regenerate TypeScript bindings post-refactor phases 1-8"
gh pr create --title "chore: regen TS bindings post-refactor" --body "Regenerate auto-generated bindings.ts after all 8 backend refactor phases."
```
